chainercvのモデルでファインチューニングしようとして死ぬ
Chainer ChainerCV Deep Learning
Lastmod: 2019-05-23 
import chainer
import chainer.links as L
import chainer.functions as F
from chainercv.links import ResNet50

class PretrainedHeadModel(chainer.Chain):
    def __init__(self,　n_classes=10):
        super(PretrainedHeadModel, self).__init__()
        with self.init_scope():
            self.base = ResNet50(pretrained_model='imagenet')
            self.pick = ['pool5']
            self.fc = L.Linear(None, n_classes)

    def forward(self, x):
        x = self.base(x)
        x = self.fc(x)
        return x

こんなふうにモデルを定義してfine-tuningを試みたが,ほとんど精度が出なかったりやたらと学習が不安定だったりして詰まった.原因は,self.base.pick = ['pool5']としてglobal average pooling直後の特徴量をfcに渡さなければいけないところが,self.pick = ['pool5']と書き間違えてsoftmax後の出力がfcに渡されたことにある. かりに正しく書いた場合は forwardにおいてx = self.base(x)[0]とかしなければならないのだが,二重の間違いを犯したために見た目上はちゃんと動いているようになってしまった. データセットによってはvalidation accuracy 0.9くらい出たり,あるい別のデータセットではtraining lossが初期値とほとんど同じまま推移することがあった.

class PretrainedHeadModel(chainer.Chain):
    def __init__(self,　n_classes=10):
        super(PretrainedHeadModel, self).__init__()
        with self.init_scope():
            self.base = ResNet50(pretrained_model='imagenet')
            self.base.pick = ['pool5']
            self.fc = L.Linear(None, n_classes)

    def forward(self, x):
        x = self.base(x)[0]
        x = self.fc(x)
        return x

これが正しいコード.
